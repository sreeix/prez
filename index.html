What is REST

Web is Rest

Resources and URI's

  • Set of exposed resources (like Customers)
  • Each resource is a standard URI
  • There is a consistent Representation for the resource

Resources can be CRUD

  • CREATE is a POST on resource (ie POST to /customers with a representation to create a resource)
  • READ is a GET on resource (ie GET to /customers/100 to get representation of the customer 100)
  • UPDATE is a PUT on resource (ie PUT to /customers/100 with the new representation of the customer 100)
  • DELETE is a DELETE on resource (ie DELETE to /customers/100 to delete the customer 100)

Standard error codes

  • 200 for success
  • 400 for failures(typically retrying won't help)
  • 500 for unknown failure(retrying may help)

Advantages of REST

  • Simple and consistent for client developers
  • WWW already supports it(like caching and Firewall access)
  • Scales very well
